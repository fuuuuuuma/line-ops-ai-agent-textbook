LINE Harness セットアップ完全チェックリスト

開発者本人による解説動画の実演をもとにした、事前準備からWebhook接続完了までのチェックリストです。つまずきやすいポイントも含めています。

  • Cloudflare・LINE公式アカウント・LINE開発者アカウントの準備手順
  • スコープのチェック漏れ、友達追加オプションのリンク忘れなど実際のハマりポイント
  • Webhook接続・LIFFエンドポイント差し替えまでの最終仕上げ
■ 事前準備（目安20分・すべて無料）

□ 1. Cloudflareアカウントを作る
  - Googleアカウント等でサインアップ
  - 「R2オブジェクトストレージ」を有効化（LINEでの画像/動画配信に使用）
    → クレジットカード登録が必要だが、10GBまでは請求0円
    → 「Continue to R2」でアクティベート完了を確認

□ 2. LINE公式アカウントを作る
  - LINEアカウント or メールアドレスで登録 → 電話番号認証
  - アカウント名・メールアドレス・業種を入力（業種は後から変更可）
  - 「チャット」→「応答設定」で 挨拶メッセージOFF・応答メッセージOFF に変更
  - 「メッセージAPI設定を開く」→利用開始に同意→プロバイダー名を設定
    → チャネルID・チャネルシークレットが発行されるので必ず控える

□ 3. LINE開発者アカウントを作る（LINE公式アカウントと連携）
  - 同じプロバイダー内に「LINEログインチャンネル」を新規作成
    地域=日本 / アプリタイプ=Webアプリ
  - ⚠️ 作成後は必ず「公開」にする（「開発中」のままだと後で正常に動かない）
  - チャネルIDを控える
  - 「LIFF」タブでアプリを追加
    サイズ=フルサイズ / エンドポイントURLはいったん仮のURL（例: https://example.com）でOK（後で差し替え）
    ⚠️ スコープは3つとも必ずチェック（「プロフィール」のチェック漏れは開発者本人も実際にハマった既知のポイント）
    友達追加オプション = アグレッシブ
    → 発行されたLIFF IDを控える
  - 「メッセージAPI設定」タブでチャネルアクセストークン（長期）を発行して控える
  - ⚠️ 見落としがちな最終設定: LINEログインチャンネルの画面を一番下までスクロールし、「友達追加オプション」で作成したLINE公式アカウントをリンクする（これを忘れると友達登録が正常に機能しない）

■ インストール（数分）

□ 4. ターミナルで実行
  npx create-line-harness@latest <プロジェクト名>
  ⚠️ プロジェクト名は英数字のみ（日本語不可）
  ⚠️ 同じCloudflareアカウントに同名で複数回デプロイすると上書きされる。複数運用するなら名前を変える

□ 5. 控えておいた4つの値を順番に入力
  チャネルシークレット → 長期アクセストークン → チャネルID → LIFF ID

□ 6. MCPサーバー設定のプロンプトが出たら「Yes」を推奨（Claude Code/Codexから自然言語で操作できるようになる）

■ 最終仕上げ（セットアップ完了後）

□ 7. LINEの応答設定に戻り、表示されたWebhook URLを貼り付けて「Webhookの利用」をON
□ 8. LIFFのエンドポイントURLを、仮のURLから実際のURLに貼り替え
  ⚠️ 貼り付け時に末尾へ余分な空白が入っていないか確認
□ 9. 「友達追加URL」（QRコード）を取得し、実機で読み取って動作確認
□ 10. 管理画面URL＋パスワード（APIキー）でログインし、アカウントが表示されることを確認
□ 11. 【重要・自動では設定されない】「友達追加時」トリガーのステップ配信シナリオを作成し、挨拶メッセージを設定する

■ 覚えておきたいこと

- ここまでの基本セットアップ・運用は Claude Code / Codex を一切使わなくても完結する
- Claude Code / Codex が本領を発揮するのは、フォーム作成やシナリオの自動生成など、もう一段進んだカスタマイズをする時

# Lステップ タグ作成自動化 Skill

## 前提（ユーザーに必ず確認する）

Lステップの「API連携」機能は2025年12月に追加された公式機能です。実行前に、ユーザーが管理画面側で以下を済ませているか確認してください。済んでいなければ先にLステップ管理画面での設定を案内する。

1. **API連携機能はプロプラン以上の有料オプションです**（🔶目安: 月額11,000円程度、Webhookを併用中の場合はさらに+5,500円程度が必要とされています。金額は変更される可能性があるため、契約中のプラン・料金は必ずLステップ公式サイト／管理画面で確認してください）。下位プラン・無料枠では利用できないため、着手前にユーザーの契約プランを確認する
2. Lステップ管理画面の「API連携」＞「認証」タブでアクセストークンを発行済みか
3. タグ付けの「エンドポイント」を作成済みか（管理画面上でJSONキー・データ型・ラベルを設定する）
4. どのイベント（例: フォーム回答・決済完了・外部システムのWebhookなど）をきっかけにタグを付けたいか

## リクエストの組み立て方（公式仕様）

Lステップの公式API連携は以下の形式に従う。ユーザーから聞いたパラメータ名・タグ名を使って、この形式でリクエストを生成する。

- 認証: `Authorization: Bearer <アクセストークン>` ヘッダー
- Content-Type: `application/json`
- メソッド: POST
- ボディ: `{"uid": "<Lステップ友だちのユーザーID>", "params": {"<JSONキー>": "<値>"}}`
- レート制限: 🔶プロプランで月5万回、秒間はどのプランも10回が目安とされています（連続実行するツールを組む場合は間隔を空ける）。最新の正確な上限は必ずLステップ公式サイトで確認してください

## 生成するコード例（Node.js版のひな形）

```javascript
async function tagLstepFriend(uid, params, endpointUrl, accessToken) {
  const res = await fetch(endpointUrl, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ uid, params }),
  });
  if (!res.ok) throw new Error(`Lステップ API error: ${res.status}`);
  return res.json();
}
```

エンドポイントURLはLステップ管理画面でユーザーが作成したエンドポイントごとに個別発行されるため、必ずユーザー本人に確認して埋め込む。存在しないURLを推測で書かない。

## 会話の進め方

1. 「どんなきっかけで、どんなタグを付けたいですか？」を聞く（例: 「決済完了webhookを受けたらVIPタグを付けたい」）
2. 上記の前提が未設定なら、Lステップ管理画面での設定手順（エンドポイント作成→パラメータ定義→トークン取得）を案内する
3. 設定済みなら、エンドポイントURL・アクセストークン（環境変数での管理を推奨）・パラメータ名を聞き取り、上記のひな形にはめ込んだコードを生成する
4. 生成したコードは必ずユーザーにテスト実行してもらい、実際にタグが付くことを確認してから本番導入するよう伝える

## 注意事項

- アクセストークンは環境変数（`.env`等）で管理し、コードに直書きしない
- 存在しないエンドポイントURLや仕様を推測で生成しない。不明な点は必ずLステップ公式リファレンスをユーザーに確認してもらう

# LINE Harness運用Skills集

LINE Harness MCPサーバーの各ツールを、目的別の会話パターンに対応させたガイドです。ユーザーの依頼文から該当パターンを判断し、対応するMCPツールを呼び出してください。

**重要な前提**: 以下に登場するツール名・パラメータ名は、本資料の執筆環境で実際にLINE Harness MCPサーバーに接続して確認した2026年7月時点（v0.14.1相当）の仕様です。ただしLINE Harnessは開発速度が速く、バージョンアップで変わる可能性があります。**必ず実行前にMCPのツール一覧（tools/list等）で正式名称・パラメータを確認してから使ってください。** 一致しない場合はツール一覧を優先し、本Skillの記述を鵜呑みにしないでください。

## パターン1: 友だち追加時の自動シナリオを作る
依頼例: 「友だち追加した人に3日間のウェルカムシナリオを流して」
→ `create_scenario`（name, triggerType: friend_add, steps: [{delay, type, content}, ...]）で、ユーザーが指定した本数・間隔のstepsを組み立てる。delayは「0m」（即時）「30m」（30分後）「24h」（24時間後）のような形式で指定する。typeは"text"または"flex"。

## パターン2: タグに応じたシナリオを作る
依頼例: 「VIPタグを付けた人にフォローアップシナリオを流して」
→ 先に `manage_tags`（action: list）でタグの存在を確認し、無ければ `manage_tags`（action: create, tagName, tagColor） → `create_scenario`（name, triggerType: tag_added, triggerTagId: 該当タグID, steps: [...]）

## パターン3: セグメント配信をする
依頼例: 「先月申込んだ人にセミナーの案内を配信して」
→ `list_crm_objects`（objectType: tags）等で該当タグ/条件を確認したうえで、以下のいずれかで実行する。
  - 全員/タグ単位の配信は `broadcast`（targetType: all または tag, targetTagId, messageType, messageContent）で即時送信、または `manage_broadcasts`（action: create_draft → 内容確認 → action: send）で下書きしてから送る
  - 条件を組み合わせたセグメント配信は `broadcast`（targetType: segment, segmentConditions: {operator, rules:[...]}）、または `manage_broadcasts`（action: send_to_segment, segmentConditions）※segment配信には下書き保存が無く即時送信になる点に注意
  - **配信は取り消せないため、実際に送信する前に、対象人数・メッセージ内容を必ずユーザーに見せて確認を取る**

## パターン4: クリック解析・コンバージョンレポートをまとめる
依頼例: 「今週のトラッキングリンクのクリック数を教えて」
→ `list_crm_objects`（objectType: tracked_links）で対象リンクのIDを特定し、`get_link_clicks`（linkId）で集計。広告連携の成果を聞かれたら`manage_ad_platforms`（action: list）でplatformIdを確認→`get_conversion_logs`（platformId）で送信ログを確認

## パターン5: 広告プラットフォーム連携をセットアップする
依頼例: 「Meta広告のコンバージョン計測を設定して」
→ `manage_ad_platforms`（action: create, name: meta, config: {pixel_id, access_token}）。X/Google/TikTokの場合はconfigの中身が変わる（例: Googleは{customer_id, conversion_action_id, oauth_token}）ので、実行前にツールのパラメータ説明を確認する。アクセストークン等の秘匿情報は必ずユーザー本人から直接受け取り、会話ログや生成物に平文で残さない

## パターン6: リッチメニューを作る
依頼例: 「メニューに問い合わせボタンを追加したい」
→ `create_rich_menu`（name, areas: JSON文字列で[{bounds:{x,y,width,height}, action:{type, uri/text/data}}]）で組み立てる。画像はimageData（base64）で渡すか、既存メニューの一覧確認は`manage_rich_menus`（action: list）で行う。

## 共通の注意事項
- **配信・広告連携・友だち情報の変更など「取り消せない/外部に影響する」操作の前には、必ず内容をユーザーに見せて実行の承認を得る**
- 存在しないタグID・シナリオID・アカウントIDを推測で使わない。必ず`list`系ツール（`list_crm_objects`, `manage_tags`(list), `list_friends`等）で実在確認してから操作する
- 個別の友だちの実データ（氏名・会話内容等）を資料や生成物に転記しない
- **本Skillに書かれたツール名・パラメータ名は執筆時点の実接続確認に基づく例であり、絶対の仕様書ではない。実行前に必ずツール一覧で正式名称・必須パラメータを確認すること**

LINE運用プロンプトまとめ — Lステップ×LINE Harness

コピペで使える自然言語プロンプト集です。ご自身の環境（Lステップの外部連携ツール、またはLINE Harness接続済みのClaude Code/Codex）に合わせてお使いください。

  • LINE Harness × Claude Code/Codexにそのまま話しかけられるプロンプト8本
  • Lステップ × 外部AI連携を検討するときの依頼文4本
  • 両方共通の「まず小さく試す」心構えつき
■ LINE Harness × Claude Code/Codex（MCP接続済み・自然言語でそのまま話しかける）

- 「友だち数と、直近のシナリオ・配信の状況を教えて」
- 「来週のセミナー参加者に、3日前と前日にリマインドを送るシナリオを作って」
- 「友だちのタグ分布を集計して、フォローが手薄なセグメントを教えて」
- 「フォームを作って、回答した人にVIPタグを付けてフォローシナリオを始めて」
- 「先週作ったトラッキングリンクのクリック数を教えて」
- 「Meta広告のコンバージョン計測を設定したい」
- 「友だち追加時に流れるウェルカムメッセージを3通、間隔を空けて設定して」
- 「今のリッチメニューの構成を教えて。問い合わせボタンを追加したい」

■ Lステップ × 外部AI（Webhook/API連携を組むとき、makeやDify等のノーコードツール・開発者への依頼文として）

- 「Lステップの友だち登録Webhookを受けたら、OpenAI APIで簡単な質問に自動応答するボットを作りたい。実装時間の目安と手順を教えて」
- 「業種特化のQ&Aボットを作りたい。Difyのナレッジベースと連携する構成を教えて」
- 「Lステップの公式API連携機能を使って、決済完了イベントをきっかけに友だちへ自動でタグを付けたい。エンドポイント設計からリクエスト例まで教えて」
- 「Lステップで集めた友だち情報を、外部システム（スプレッドシート等）と同期する仕組みを組みたい」

■ 使い方の心構え（両方共通）

- 最初の1回は必ず小さい範囲（テスト用の1アカウント・少人数のセグメント）で試してから本番に反映する
- 配信・タグ変更・広告連携などは「取り消せない/実際に相手に届く」操作なので、実行前に内容を必ず自分の目で確認する
