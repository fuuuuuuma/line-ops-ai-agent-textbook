## 1. LINE運用は、もう避けて通れないインフラである

結論から言います。何か商品を売るにしても、自分で何かサービスを始めるにしても、**LINE公式アカウントの構築・運用は、もはや避けて通れないインフラ**になっています。

大げさな話ではありません。LINEヤフー株式会社は2026年1月29日、国内月間利用者数が2025年12月末時点で**1億ユーザーを突破した**と正式に発表しています。日本の人口を考えると、ほぼ「大人はみんな使っている」水準です。

しかも、この数字は若年層に偏っているわけではありません。モビルス株式会社が実施した「消費者のLINE公式アカウント利用実態調査2025」によると、**60代以上のLINE利用率は約7割に到達し、この5年で1割増加した**と報告されています。「若い人向けのSNS」というイメージで語られがちなSNSマーケティングの世界で、LINEだけは世代を問わず使われている数少ないチャネルなのです。

さらに重要なのが、消費者側の「問い合わせ意欲」です。同調査では、**全年代の6割超がチャットでの企業への問い合わせを希望しており、20〜30代に限れば7割に達する**と報告されています。電話でもメールでもなく「LINEで気軽に聞きたい」というニーズが、すでに消費者側にできあがっているということです。

この土台があるからこそ、LINE公式アカウントは単なる「お知らせ配信ツール」ではなく、次のような使われ方をするようになっています。

- 予約受付・来店誘導（飲食店・サロン・クリニックなど）
- 個別相談・カウンセリングの窓口（士業・コンサル・スクール系）
- 商品購入・決済までを完結させる導線（EC・情報商材）
- 会員限定コンテンツの配信（コミュニティ・サブスク）

つまり「LINE公式アカウントを持っているだけ」では、もう不十分です。友だちをセグメントして配信したり、行動履歴に応じてメッセージを出し分けたりといった、いわゆる「マーケティングオートメーション（MA）」的な運用ができて初めて、LINEというインフラを使いこなせたと言えます。何かを売る立場でも、何かのサービスを立ち上げる立場でも、この構築・運用という工程からは逃げられません。

## 2. 自分でやるには大変、代行を仕事にするにも大変

では、このLINE運用の構築を自分の手だけでやろうとするとどうなるか。これが、率直に言って**かなり大変**です。

LINE公式アカウント単体の管理画面は、実はそこまで高機能ではありません。友だち追加のあいさつメッセージ1つとっても、「どのタイミングで」「誰に」「何を」送るかを設計しなければならず、シナリオ配信・セグメント配信・行動分析といった一歩進んだ運用をしようとすると、途端に設計・構築の手間が跳ね上がります。時間もかかるし、外部ツールを使うなら当然お金もかかる。個人や中小企業が片手間でやるには、想像以上に重い作業です。

だったら「これを仕事にしてしまえばいいのでは」と考える人もいるでしょう。実際、「Lステップ構築代行」のような形でこの構築作業を請け負うビジネスは、すでに一つの職能として成立しています。需要は高く、単価も悪くありません。公式な市場統計は見当たりませんが、X（旧Twitter）上の実務者の言及ベースでは、次のような相場感がよく語られています。

| 案件の種類 | 単価の目安（X上の実務者の言及ベース） |
|---|---|
| シンプル構築（あいさつメッセージ＋簡単な自動応答など） | 1.5万〜5万円 |
| 標準構築（シナリオ配信・セグメント配信・LP連携込み） | 5万〜30万円 |
| 本格構築（複数シナリオ・戦略設計込み） | 30万〜80万円以上 |
| 月額運用代行・保守 | 5万〜30万円／月 |

この表はあくまで「X上でよく見かける言及の幅」であり、公的な調査結果ではありません。実際の相場は地域・業種・代行者の実績によって大きく変動しますが、それでも「需要も単価もそれなりに高い仕事」であることは間違いなさそうです。

ただし、単価が良いということは、それだけ「大変な仕事」だということの裏返しでもあります。タグ設計・シナリオ分岐・流入経路の整理といった作業を、案件ごとに一つひとつ手作業で組み立てていくのは、決して楽な仕事ではありません。しかも、単純な設定作業そのものはAIによって急速にコモディティ化（誰でも・安く・早くできるもの）が進みつつあり、代行者側の価値の源泉は「設定を代行できるかどうか」から「なぜこの導線で成果が出るのかを設計できるかどうか」、つまり戦略設計力や業種特化のノウハウへと移りつつあります。

自分でやるにも大変、仕事として請け負うにも大変。この「大変さ」こそが、次章で紹介するAI活用の出発点です。

## 3. だからAIを使う——今回の2本立て

自分でやるにも大変、代行を仕事にするにも大変。だからこそ、AIを使います。

Claude Code（チャンネルによってはCodex）を使えば、このLINE運用の構築・運用作業をかなり時短・自動化できます。単純作業をAIに肩代わりさせ、人間は戦略設計や最終確認だけに集中する——そんな役割分担が、もう現実的な選択肢になっています。

今回は、この「AI×LINE運用」というテーマを、2本立てで解説します。

- **① 国内で最も知名度の高いLステップを、AIで自動化する話**：業界最大手級と言われるLINEマーケティングツール「Lステップ」の、あの地道な単純作業をClaude Codeの「Computer Use」機能でどう自動化するか
- **② オープンソースの「LINE Harness」の導入方法・使い方**：無料・オープンソースで公開されている新興ツール「LINE Harness」を、実際にどう導入し、どう使うか

そして、実際に手を動かして試せるように、この記事にはプレゼントも用意しています。中身の詳細は最後にまとめてお渡ししますが、実際に導入する際にはぜひ使ってみてください。

## 4. Lステップ編

### 4-1. Lステップとは何か

**Lステップ**は、LINE公式アカウントと連携して使う、LINEマーケティングオートメーションツールです。国内で最も知名度の高いツールの一つで、「LINE構築＝Lステップ構築」と同義で語られることも珍しくありません。

安心して使える理由の一つが、運営体制です。Lステップは公式・大手企業が運営しているサービスであり、個人が趣味で作った野良ツールではありません。実際、Lステップの公式サイトには「※『Lステップ』はLINE公式アカウントのAPIを拡張したツールであり、弊社とLINEヤフー株式会社との関係はございません」という注記が明記されています。これは裏を返せば、独立系の企業がLINE公式のAPIをきちんと拡張する形で作り込んでいる、公式サポート体制のあるサービスだということです。国内屈指の実績と知名度を持つツールとして、初めてLINE運用を本格化させる人にとって安全・安心な選択肢の筆頭になっています。

料金は複数のプランが用意されており、目安として**月額5,000円〜32,780円程度**（プロプランまで含む）のレンジで語られることが多いです。ただし価格改定はしばしば行われるため、最新の正確な価格は必ず公式サイトで確認してください。

導入の流れは、おおむね次の3ステップです。

1. **申込・プラン選択**：Lステップに申し込み、利用するプランを選び、クレジットカードで認証する
2. **チャネル情報の連携**：LINE Developersコンソールで「チャネルID」「チャネルシークレット」を取得し、Lステップの管理画面に登録する
3. **初期設定**：友だち追加時に送る「あいさつメッセージ」など、基本の応答を設定する

Lステップを使うと、次のようなことがすべてGUI（管理画面のクリック操作）だけでできるようになります。

- **シナリオ配信**：友だち追加からの経過日数に応じて、決まったメッセージを自動配信する
- **セグメント配信**：「タグ」や「行動」に応じて、送る相手を絞り込んで配信する
- **行動分析**：どのメッセージが開封されたか、どのリンクがクリックされたかを分析する
- **回答フォーム・アンケート**：LINE上で完結する入力フォームを作る

非エンジニアでも管理画面の中で完結できる、というのがLステップ最大の強みです。ですが、これらの機能を「使える」ことと「使いこなせる」ことの間には、大きな溝があります。次の項目で、その正体を具体的に見ていきます。

### 4-2. Lステップの「単純作業」という課題

Lステップは非常に高機能なぶん、仕様上、実際の運用に入る前に大量の"下ごしらえ"が必要になります。具体的には、次のような作業です。

- **タグの大量作成**：顧客の属性・行動ごとに、細かく分類したタグを何十個、何百個と用意する必要があります。「セミナー参加済み」「商品A購入」「メルマガ未読3回」など、セグメント配信の精度を上げようとすればするほど、タグの数はどんどん増えていきます
- **流入経路の設定**：広告・SNS・紹介など、どこから来た友だちかを識別するための経路を、流入元の数だけ個別に作る必要があります。広告媒体を増やすたびに、この作業も比例して増えます
- **メッセージテンプレートの作成**：似たような文面のテンプレートを、ターゲットや配信タイミングごとに何パターンも用意しなければなりません

これらに共通しているのは、「Lステップの管理画面を開いて、フォームに入力して、保存する」という一連の操作を、何十回、何百回と繰り返す単純作業だということです。1件1件の作業自体は簡単でも、それが100件、200件と積み重なると、丸一日、二日をこの"コピー＆ペーストの繰り返し"だけで溶かしてしまうことになります。しかも、この作業には創造性も戦略性もほとんど求められません。ただひたすら同じ手順を繰り返すだけの、疲弊するタイプの仕事です。

### 4-3. Computer Use（Claude Codeのブラウザ操作機能）で自動化する、という解決策

ここで登場するのが、Claude Codeの「Computer Use」という機能です。

Computer Useとは、Claude Codeのブラウザ操作機能のことで、画面を見ながらマウス・キーボード操作を自動化する仕組みです。人間が管理画面を見ながらクリックし、入力し、保存するのとまったく同じことを、AIエージェントが代わりにやってくれます。実際、本資料の執筆環境でもこのComputer Use機能を実際に使用できることを確認済みで、今回の調査そのものにもこの仕組みを活用しています。なお、この機能はあくまでClaude Code側のものであり、Codexに同名・同等の機能があるかどうかは未確認です。

前の項目で挙げたような「同じ作業を何度も繰り返す」単純作業は、まさにComputer Useが最も得意とする領域です。たとえば、次のように頼むだけで済むようになります。

```
> このリストにある20個のタグを、Lステップの管理画面で全部作ってください。
> タグ名はスプレッドシートのA列を上から順番に使ってください。
```

```
> このスプレッドシートに書いてある流入経路を、
> Lステップの「流入経路」設定画面に全部登録してください。
```

人がクリックし続ける代わりに、AIが同じ操作を黙々と繰り返してくれる。これだけで、Lステップ運用の中でもっとも消耗するパートを丸ごと肩代わりさせることができます。

Computer Useの活用先は、この"単純作業の自動化"だけではありません。他にも、こんなところにAIを使えます。note「Lステップ連携ラボ」（note.com/glad_walrus6151）では、LステップのWebhook機能を使ったAIチャットボット構築の実例がいくつも紹介されています。

Webhookというのは、日常語で言えば「LINEで何かが起きたら、自動で別の場所に知らせる仕組み」のことです。この仕組みそのものはLステップ側が用意していますが、そこから先の「受け取ったメッセージにどう返すか」を担当するのが、ChatGPTやDifyといった外部のAIツールです。「Lステップ連携ラボ」で紹介されている実例には、次のようなものがあります。

- **make（ノーコード自動化ツール）＋ OpenAI APIによるシンプルな1問1答ボット**：料理レシピを提案するボットなどが実装例として紹介されており、実装時間の目安は30〜90分程度とされています
- **make ＋ Dify ＋ ナレッジベースによる、業種特化の専門回答ボット**：データベース（ナレッジベース）と連携することで、より専門性の高い回答ができるようになります

同じく「Lステップ連携ラボ」では、就活支援AIチャットボット（ES自動作成・添削、適性診断）や、決済連携での自動収益化ガイドといったタイトルの実例記事も確認できます。これらはタイトルからの推定であり本文までは読み込めていない点にはご留意ください。それでも、Lステップ×AIの組み合わせが「単純作業の自動化」だけでなく、「顧客対応そのものの自動化」にまで広がっていることが伝わってくるはずです。

### 4-4. 難易度が高い連携のステップバイステップ

紹介した事例の中で、特に難易度が高いのが「make ＋ Dify ＋ ナレッジベース」による専門知識対応型のボットです。ここでは、この構成をどう組み立てていくかを、ステップバイステップで整理します。

1. **Webhookを受け取る土台を作る**：makeでLステップからのWebhookを受信するシナリオを作成します。まずはLステップの「友だちがメッセージを送ってきたら発火する」設定を確認するところから始めます
2. **Difyにナレッジベースを用意する**：業種特有のFAQやマニュアルをテキスト化し、Difyのナレッジベース機能にアップロードします。ここで用意する情報の質が、そのままボットの回答品質に直結します
3. **Difyのアプリを組む**：アップロードしたナレッジベースを参照する形で、質問応答用のアプリ（チャットボット）をDify上に構築します
4. **makeでLステップとDifyをつなぐ**：Webhookで受信したメッセージをDifyのAPIに渡し、返ってきた回答をLステップ経由でLINEに返信するフローを組みます
5. **小さくテストする**：いきなり全顧客に公開せず、社内やテストアカウントで質疑応答の精度を確認してから本番公開します

この一連の設定作業も、実はComputer Useやコーディングエージェントの支援を受けながら進めることができます。「DifyのナレッジベースにこのマニュアルPDFをアップロードして」「makeでこういうフローを組みたいので手順を教えて」といった形でAIエージェントに相談しながら進めれば、初めての人でも迷わず組み立てられるはずです。パターン1（make＋OpenAI APIのシンプル型）よりは設定の難易度は上がりますが、それに見合うだけの専門性の高い回答が実現できる構成です。

### 4-5. Lステップの2つのデメリット

ここまでLステップの良さと、AIによる自動化の可能性を紹介してきました。ですが、Lステップにはどうしても避けられない2つのデメリットがあります。

1. **柔軟性が低い**：Lステップは公式のクローズドなツールです。安全・安心の裏返しとして、機能追加は運営会社の開発スケジュールに依存し、かゆいところに手が届くようなカスタマイズは基本的にできません。「自分たちの業種に合わせて、この部分だけ仕様を変えたい」と思っても、それを自由に実現する手段は用意されていないのです
2. **価格が高い**：月額5,000円〜32,780円程度かかり、さらにオプション機能を使うと追加費用が発生することもあります。実際、2025年12月には公式のAPI連携機能が新たに追加されましたが、この機能を使うには月額11,000円程度、Webhookを併用する場合はさらに5,500円程度の追加費用がかかるとされています。運用が本格化するほど、じわじわとコストが積み上がっていく構造です

「安全・安心だけど、自由が利かなくて、お金もかかる」——このLステップの弱点を正面から解消しようとしているのが、次に紹介するオープンソースツール「LINE Harness」です。

## 5. LINE Harness編

### 5-1. LINE Harnessとは何か

**LINE Harness**は、LINE公式アカウント運用のための**完全オープンソースCRM**です。「The Harness」というプロジェクトシリーズの第一弾として公開されており、第二弾の「X Harness」、第三弾の「IG Harness」もすでに公開済みです。最新のGitHubリポジトリでは「次はどの業界を0円にしようかな」という第四弾の予告まで確認できます。

開発者は野田修一氏（X: @ai_shunoda、GitHub: @Shudesu）です。開発の原体験は、既存のLINE拡張ツールに1アカウントあたり月5万円、年間にすると数百万円単位を支払っていた経験だったと語られています。この「高すぎるコスト構造」への違和感が、ツールを自作してオープンソースで無料公開するという発想につながったようです。

「ハーネス」という名前の由来も興味深いものです。犬用ハーネスのように、AIが暴走しないための「外枠」という意味が込められています。中身のAIモデル（現在はClaude Code）は今後入れ替わっていく可能性がありますが、ハーネスというオープンソースの枠組み自体は存続し続ける、という設計思想です。

料金は**完全無料**です。Cloudflare Workers上で動作し、友だち5,000人までは実質0円で運用できます。5,000人を超えるとCloudflareの利用料が発生しますが、これはインフラを提供するCloudflareへの実費であり、LINE Harness自体への課金ではありません。

実際にGitHubリポジトリを確認したところ（2026年7月5日時点）、スター515、フォーク301、コミット175件、リリース17件（最新v0.15.0）、Issue15件・PR12件という活発な開発状況が見て取れます。READMEは日本語・英語・スペイン語・韓国語・中国語の5言語に対応しており、`AGENTS.md`というファイルも同梱されていて「conversation inbox MCP（v0.11.0）」のような機能追加が続いていることが分かります。開発ペースの速さは、個人開発のオープンソースプロジェクトとしては驚くべき水準です。

公式サイト（shudesu.github.io/line-harness-oss/）のデザインも、このツールの「攻めた」姿勢を表しています。黒〜濃緑のグラデーション背景に、ミントグリーンの「LINE Harness」ロゴ。そして「L社やU社の上位互換を、なぜ無料で公開するのか。」という見出しの下に、大きく「0円」という数字が表示されています。既存の有料ツールを名指しで意識した、かなり強気なメッセージです。ページの下部には開発者プロフィールカードもあり、野田修一氏本人が「LINE Harness開発者 / AI Engineer」としてプロフィールを掲載する形で紹介されています。

さらにサイトの下部には「LINE Harnessは、始まりに過ぎません。」というロードマップ見出しがあり、その下に「LINE Harness（公開中）／X Harness（公開中）／IG Harness（公開中）／???Harness（次弾）」という4枚のカードが並んでいます。LINEだけでなく、SNS運用全体をオープンソース＋AIエージェントで置き換えていく——そんな野心的な構想がうかがえます。

LINE Harnessの最大の特徴は、**MCPサーバーを標準搭載**していることです。MCP（Model Context Protocol）とは、AIエージェントが外部のツールを「道具」として直接操作できるようにする、Anthropicが提唱するオープンな規格のことです。これにより、Claude CodeのようなAIエージェントから、LINE公式アカウントのほぼすべての機能を自然言語だけで操作できるようになります。

開発者本人も、この点をXで強く打ち出しています。2026年3月26日午後4:11に投稿されたポスト（1.4万表示・返信3・リポスト20・いいね143・ブックマーク128）では、次のように紹介されています。

> 「LINE Harness v0.4.0 — MCPサーバー対応🎉 Claude CodeからLINE公式アカウントを完全操作できるようになった。『友だち数教えて』『VIPに配信して』『クリック数見せて』─全部自然言語でいける。16ツール、管理画面いらず。
> - URL自動追跡（送信リンクを自動トラッキング化、誰が押したか特定）
> - テキスト→Flex自動変換（URLをボタン化）
> - LIFF連携クリック計測（LINEアプリ→ユーザー特定、PC→ログイン不要）
> - MCPサーバーソースOSS公開」

管理画面を開いてクリックし続ける代わりに、チャットで「友だち数教えて」と話しかけるだけで完結する——これがLINE Harnessの最大の武器です。

### 5-2. 導入方法をステップバイステップで

LINE Harnessの導入方法を、開発者本人による実演動画で確認できた手順に沿って、ステップバイステップで整理します。事前準備を含めても、目安は**20分程度**です。

**事前準備（目安20分・すべて無料）**

1. **Cloudflareアカウントを作り、R2オブジェクトストレージを有効化する**。LINEでの画像・動画配信用のストレージとして使うため、クレジットカードの登録が必要になりますが、請求は0円、10GBまでは無料枠で足ります
2. **LINE公式アカウントを作る**。応答設定でWebhookをONにするための準備として、メッセージAPI設定を開き、チャネルID・チャネルシークレットを取得しておきます
3. **LINE開発者アカウントでLINEログインチャンネルを作成する**。ここには2つの重要な落とし穴があります。1つ目は、**作成したチャンネルは必ず「公開」状態にすること**です。「開発中」のままだと動作しません。2つ目は、LIFFアプリを追加する際、**スコープは3つ全部チェックすること**です。開発者本人も実演の中で「プロフィール」のチェックを入れ忘れて、後からバグに気づいた実例を明かしています。さらに、チャネルアクセストークン（長期）を発行し、**LINEログインチャンネルの「友だち追加オプション」で、作成したLINE公式アカウントをリンクするのを忘れないこと**。これを忘れると、友だち登録そのものが機能しなくなります

**インストール**

事前準備で控えた4つの値（チャネルID・チャネルシークレット・チャネルアクセストークン・LIFF ID）を手元に用意したら、ターミナルで次のコマンドを実行します。

```bash
npx create-line-harness@latest
```

これだけで、3ステップでのLINE Harness導入が始まります。プロジェクト名を指定したい場合は続けて英数字の名前を渡すこともできますが、必須ではありません（同じ名前で複数回デプロイすると上書きされる点だけ注意してください）。

**ターミナル操作が初めての方向けに、開発者が専用のセットアップガイドページを用意しています**（[line-crm-worker.line-crm-api.workers.dev/setup](https://line-crm-worker.line-crm-api.workers.dev/setup)）。Windows／Macそれぞれで「ターミナルの開き方」から画面つきで説明されており、「PowerShellではなくコマンドプロンプト（cmd）を使う」「パスワード入力時は画面に何も表示されない」といった、初心者がつまずきやすいポイントも先回りしてフォローされています。

コマンドを実行すると、控えておいた4つの値を順番に入力するよう求められます。途中でMCPサーバー設定を聞かれた場合は「Yes」と答えることを推奨します。

**最終仕上げ**

インストールが終わったら、次の作業で仕上げます。

- LINEの応答設定に戻り、Webhook URLを貼り付けて「Webhookの利用」をONにする
- LIFFのエンドポイントURLを、仮のURLから実際のURLに差し替える（**末尾に余分な空白が入りやすいので要注意**）
- 友だち追加URL（QRコード）を取得する
- 管理画面にログインできることを確認する
- 「友だち追加時」に発火するシナリオ（挨拶メッセージ）を手動で作成する

**もう一つ、動画公開後に開発者本人が追記で訂正した重要な設定があります**。LINEログインチャンネルの設定で「Callback URL」を登録する手順が、当初の動画では抜けていました。これを設定しないと、PCからQRコードを踏んだ人が「Invalid redirect_uri」というエラーで友だち追加できません（スマホの場合はLIFFアプリ経由で動作するため気づきにくい落とし穴です）。設定方法は次の通りです。

- LINE Developers Console → LINEログインチャンネル → 「LINEログイン設定」 → 「ウェブアプリでLINEログインを利用する」をオンにする
- 「Callback URL」欄に `https://{あなたのworker名}.workers.dev/auth/callback` を貼り付ける

なお、開発者によれば最新版の`create-line-harness@latest`ではセットアップの最後にこの案内が自動表示されるようになっているとのことです。より詳しい手順は公式Wiki（[harness-wiki.pages.dev/line](https://harness-wiki.pages.dev/line)）とGitHubリポジトリ（[github.com/Shudesu/line-harness-oss](https://github.com/Shudesu/line-harness-oss)）でも確認できます。

ここで重要な事実をお伝えしておきます。**この基本セットアップ・一般的な運用には、Claude Codeは一切使われていません**。ここまでの作業はすべて、Cloudflareやターミナル、LINEの管理画面だけで完結します。Claude Codeが必要になるのは、フォーム作成やシナリオの自動生成など、もう一段進んだカスタマイズをする時です。つまり「導入のハードルは低いが、使いこなす上でAIエージェントが本領を発揮する」という設計になっているわけです。

### 5-3. 実際に何ができるのか

導入が終わったら、実際にLINE Harnessで何ができるのかを見ていきましょう。

MCPサーバー経由でClaude Codeから自然言語操作ができる範囲は、非常に広範です。本資料の執筆環境でも実際にLINE HarnessのMCPサーバーに接続して確認したところ、次のような30種近いツールが利用可能でした。

- シナリオ管理
- 友だち管理
- タグ管理
- リッチメニュー作成
- 配信（ブロードキャスト）
- フォーム作成
- 広告プラットフォーム連携（Meta・X・Google・TikTok）
- トラッキングリンク作成
- トラフィックプール管理
- スタッフ管理

これだけの機能を、管理画面を1つずつ開くのではなく、「友だち数教えて」「VIPに配信して」「クリック数見せて」といった日常語の指示だけで動かせるのが、LINE Harnessの強みです。

実際の活用事例として印象的なのが、カスタマーサクセス（CS）の文脈での使い方です。AIコンサル業を営むれん氏（aivest-channel）が対談で語った事例では、受講生がLINEで質問すると、その人の受講履歴・過去のレッスン内容と紐づいた回答が返ってくる仕組みが構築されています。さらに、Googleカレンダー連携によるカレンダー予約機能と組み合わせることで、次のようなワークフローが実現できています。

```
面談を予約 → Google Meetで面談 → 録画 → テキスト化 → タスク抽出
→ 後から「あの時何て言ってたっけ」とAIチャットに聞くと答えてくれる
```

構築にかかった期間は、待機時間も含めて「大体1日」とのことです。ただし本人は「Claude Codeの扱い方に熟知しているエンジニア経験者だからこそ早く構築できた」と明言しており、非エンジニアが同レベルのカスタマイズを1日で行うのは、前提知識のハードルがある点には注意が必要です。

### バランスを取るための、率直な意見も

ここまでLINE Harnessの利点を中心に紹介してきましたが、資料全体のバランスを取るため、Lステップ構築を専門にしている代理店側の視点も紹介しておきます。以下はあくまで発言者個人の見解であり、断定するものではありません。発言者はLステップ構築を生業とする代理店側の当事者であるため、無料の競合サービスに対しては厳しめの評価になりやすい立場にある、という点も踏まえて読んでください。

LステップNo.1代理店を名乗る運営者からは、次のような懸念が語られています。

- LINE Harnessは1人のエンジニアが個人開発しているオープンソースであり、Lステップのような大手が運営する既存サービスに比べると、安定性の実績はまだ少ないという指摘です。自分でサーバー（Cloudflare）を用意して運用する自己責任の要素があり、Cloudflareが数ヶ月に1回程度落ちることがあり、そのタイミングで配信が止まるリスクも指摘されています
- Claude CodeのSkillsやプロンプトの雛形ファイルに業務ノウハウが集約されるため、その環境が万一侵害されると情報がまとめて流出するリスクがあるという指摘もあります
- 「クライアントワークで他社に勧めるのは（セキュリティ面で）避けたい、自社利用なら自己責任で」という立場の意見もあります

数字や機能だけを見るとLINE Harnessが圧倒的に有利に見えるかもしれませんが、実際には「どちらが優れているか」ではなく「自分の状況にどちらが合っているか」で考えるべきでしょう。非エンジニアで手厚いサポートを重視するならLステップ、コストを抑えたい・技術力があり自由度を求めるならLINE Harness、というのが大まかな目安になりそうです。

重要なのは、以前は「Lステップ一択」に近かった市場に、無料かつAIエージェントとの相性がよい選択肢が登場した、という構図の変化そのものです。

## 6. まとめ——実際に導入するときは、このプレゼントを使ってください

ここまで、Lステップ編とLINE Harness編の2本立てで、AIエージェントによるLINE運用の自動化を見てきました。

- **Lステップ**：公式・大手企業が運営する安全・安心の定番ツール。タグ・流入経路・テンプレートの大量作成という単純作業は、Computer Useで自動化できる。ただし柔軟性の低さと価格の高さという2つのデメリットがある
- **LINE Harness**：そのデメリットを解消するオープンソースの選択肢。完全無料、MCPサーバー標準搭載でClaude Codeから自然言語で運用できる。その分、技術的なセットアップと自己責任の比重が高い

どちらか一方が絶対的な正解というわけではありません。すでにLステップで運用中なら、無理に乗り換える必要はなく、まずはComputer UseやWebhook連携でのAI活用から試すのが現実的です。これから新しくLINE運用を始める、コストをかけずに試したいという場合は、LINE Harnessの無料枠から触ってみるという選択肢が増えたことになります。
