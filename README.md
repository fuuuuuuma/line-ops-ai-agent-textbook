## 1. LINE運用の重要性 — なぜ今LINEなのか

「うちはLINE公式アカウントさえあれば十分」——そう考えている個人事業主や中小企業の担当者は、実はかなり理にかなっています。LINEヤフー株式会社は2026年1月29日、**国内月間利用者数が2025年12月末時点で1億ユーザーを突破した**と正式発表しました ✅（本資料が当初参照した2025年6月23日更新の媒体資料では9,800万人という数値でしたが、その後の公式発表を踏まえ最新値に更新しています）。日本の人口を考えると、ほぼ「大人はみんな使っている」水準のインフラです。

この数字の厚みは年代別に見るとさらに際立ちます。モビルス株式会社が2025年6月5〜11日に実施した「消費者のLINE公式アカウント利用実態調査2025」（有効回答655名）によると、**60代以上のLINE利用率は約7割に到達し、5年で1割増加** ✅ しています。「若年層のSNS」というイメージで語られがちなSNSマーケティングの世界で、LINEだけは世代を問わず使われている数少ないチャネルだということです。

さらに重要なのが、消費者側の「問い合わせ意欲」です。同調査では**全年代の6割超がチャットでの企業への問い合わせを希望しており、20〜30代に限れば7割に達する** ✅ と報告されています。つまり、電話でもメールでもなく「LINEで気軽に聞きたい」というニーズが、すでに消費者側にできあがっているのです。

この土台があるからこそ、LINE公式アカウントは単なる「お知らせ配信ツール」ではなく、次のような使われ方をするようになっています。

- 予約受付・来店誘導（飲食店・サロン・クリニックなど）
- 個別相談・カウンセリングの窓口（士業・コンサル・スクール系）
- 商品購入・決済までを完結させる導線（EC・情報商材）
- 会員限定コンテンツの配信（コミュニティ・サブスク）

ここで問題になるのが「LINE公式アカウントの管理画面そのものは、実はそこまで高機能ではない」という点です。友だちをセグメントして配信したり、行動履歴に応じてシナリオを分岐させたりといった「マーケティングオートメーション（MA）」的な動きは、標準機能だけではかなり難しい。ここに、これから紹介する「Lステップ」のような拡張ツールが生まれる余地があります。

そして2026年に入り、この構図に新しい変数が加わりました。**AIエージェントが、LINE運用そのものを「話しかけるだけ」で操作できるようになった**ということです。本記事では、この変化を「有料の定番ツール（Lステップ）」と「無料のオープンソースツール（LINE Harness）」という2つの軸から、初心者にも分かる言葉で解説していきます。

## 2. LINE構築はなぜ大変なのか — 市場感とマネタイズの実態

LINE公式アカウントを「ただ作る」のは無料ですぐできます。ですが、前章で触れたような「シナリオ配信」「セグメント配信」「行動分析」まで作り込もうとすると、途端にハードルが上がります。友だち追加のあいさつメッセージ1つとっても、「どのタイミングで」「誰に」「何を」送るかを設計しなければならず、これは立派な設計スキルです。

この「作り込みの大変さ」を裏側から見ると、「LINE構築代行」というビジネスが成立する理由が見えてきます。公式な市場規模の統計は見つかっていません ⚠️ が、LINE公式アカウントの利用者基盤の大きさ（前章の1億人という利用者数）を背景に、構築代行市場は一定の規模で存在していると推定されています ⚠️。

具体的な金額感については、公式統計ではなく**X（旧Twitter）上の実務者の言及ベースの相場感** 🔶 として、次のような幅で語られることが多いです。

| 案件の種類 | 単価の目安（🔶実務者の相場感） |
|---|---|
| シンプル構築（あいさつメッセージ＋簡単な自動応答など） | 1.5万〜5万円 |
| 標準構築（シナリオ配信・セグメント配信・LP連携込み） | 5万〜30万円 |
| 本格構築（複数シナリオ・戦略設計込み） | 30万〜80万円以上 |
| 月額運用代行・保守 | 5万〜30万円／月 |

この表はあくまで「X上でよく見かける言及の幅」であり、公的な調査結果ではない点に注意してください 🔶。実際の相場は地域・業種・代行者の実績によって大きく変動します。

もう一つ押さえておきたいのが、**この市場そのものが構造変化の途中にある**ということです。「あいさつメッセージを設定する」「シナリオを組む」といった純粋な"設定作業"は、AIによって急速にコモディティ化（誰でも・安く・早くできるもの）が進んでいます 🔶。実際、後述するLステップのAI連携やLINE HarnessのMCP連携は、まさにこの"設定作業の自動化"を実現するものです。

その結果、構築代行者側の差別化ポイントは、「設定を代行できるかどうか」から「なぜこの導線で成果が出るのかを設計できるかどうか」——つまり**戦略設計力・業種特化・運用ノウハウ**へと移りつつあります 🔶。「友だち数が◯倍になった」「問い合わせが◯倍になった」といった成功事例が語られることもありますが、これらは個別の事例ベースの話であり、一般化・保証はできない点も明記しておきます ⚠️。

この章の要点をまとめると、次のようになります。

- LINE構築には設計スキルが必要で、それゆえに代行ビジネスが成立してきた ⚠️（市場規模は未確認）
- 単価感はX上の実務者の相場感ベースで数万円〜数十万円という幅がある 🔶
- 「設定」はAIでコモディティ化が進み、価値の源泉は戦略設計・業種特化にシフトしている 🔶

そして、この「設定のコモディティ化」を体現しているのが、次章から紹介するLステップとLINE HarnessそれぞれのAI連携です。

## 3. Lステップとは

**Lステップ**は、LINE公式アカウントと連携して使う、有料のLINEマーケティングオートメーションツールです。シナリオ配信・セグメント配信・行動分析といった、LINE公式アカウント単体では難しい機能を追加できます ✅。国内のLINEマーケティング界隈では最も知名度の高いツールの一つで、「LINE構築＝Lステップ構築」と同義で語られることも珍しくありません。

### 料金感

料金プランは複数用意されており、目安として**月額5,000円〜32,780円程度** 🔶 のレンジで語られることが多いです。ただし価格改定はしばしば行われるため、最新の正確な価格は必ず公式サイトで確認してください 🔶。本記事では価格を断定する意図はなく、あくまで「有料ツールである」という位置づけの理解に留めてください。

### 導入の一般的な流れ

Lステップの導入は、おおむね次の3ステップで進みます ✅。

1. **申込・プラン選択**: Lステップに申し込み、利用するプランを選び、クレジットカードで認証する
2. **チャネル情報の連携**: LINE Developersコンソールで「チャネルID」「チャネルシークレット」を取得し、Lステップの管理画面に登録する
3. **初期設定**: 友だち追加時に送る「あいさつメッセージ」など、基本の応答を設定する

この3ステップの中で、非エンジニアがつまずきやすいのが2番目の「チャネル情報の連携」です。LINE Developersコンソールという、ふだん馴染みのない管理画面を操作する必要があるため、ここで挫折してしまう人が一定数います。逆に言えば、この最初のハードルさえ越えてしまえば、あとは管理画面の中でGUI操作を進めていく形になります。

### Lステップでできること（管理画面ベース）

- **シナリオ配信**: 友だち追加からの経過日数に応じて、決まったメッセージを自動配信する
- **セグメント配信**: 「タグ」や「行動」に応じて、送る相手を絞り込んで配信する
- **行動分析**: どのメッセージが開封されたか、どのリンクがクリックされたかを分析する
- **回答フォーム・アンケート**: LINE上で完結する入力フォームを作る

これらはすべてGUI（管理画面のクリック操作）で完結するように設計されており、「非エンジニアでも使える」ことが最大の強みです。ただし、複雑なシナリオを設計しようとすると、条件分岐やタグ設計の組み立てそのものに専門知識が必要になってくる、というのが実情です。だからこそ、前章で触れた「構築代行」というビジネスが成立してきました。

そして次章では、この「Lステップの設定・運用」をAIエージェントに手伝わせる方法を紹介します。

## 4. LステップでAIを使う方法

Lステップ自体には、2026年7月時点で公式のAIエージェント機能は搭載されていません 🔶。ですが、Lステップの「Webhook」という仕組みを使うことで、外部のAIツールと連携させることができます。この分野で発信を続けているのが、note「Lステップ連携ラボ」（note.com/glad_walrus6151）です ✅。

### Webhook連携とは何か（悩み→こう頼む、の前提知識）

まず「Webhook」という言葉に身構える必要はありません。日常語で言えば、「LINEで何かが起きたら、自動で別の場所に知らせる仕組み」のことです ✅。たとえば「お客さんがLINEでメッセージを送ってきたら、自動でスプレッドシートに記録する」「LINEで予約が入ったら、自動でSlackに通知する」といった動きが、これにあたります。

Webhookの仕組みそのものはLステップ側が用意している機能で、そこから先の「受け取ったメッセージにどう返すか」を担当するのが、ChatGPTやDifyなどのAIツールです。

### パターン1: リアルタイム応答型（シンプル）

一つ目のパターンは、**make（旧Integromat）＋ OpenAI API** を組み合わせる方法です ✅。「Lステップ連携ラボ」で紹介されている例では、料理レシピを提案するボットが実装されており、実装時間の目安は**30〜90分程度** ✅ とされています。

このパターンのメリットは「設定が簡単、コストが安い」ことです ✅。プログラミング知識がなくても、makeの画面上でパーツをつなぐようにフローを組めるため、Lステップ導入検討者や非エンジニアでも実装できると強調されています ✅。

悩み→こう頼む、で整理すると次のようになります。

| こんな悩み | こう組む（パターン1） |
|---|---|
| 「よくある質問に自動で答えたい」 | make＋OpenAI APIでWebhook受信→AI応答→LINE返信の3ステップを組む |
| 「実装に時間をかけたくない」 | このパターンなら30〜90分程度が目安 ✅ |
| 「まずは小さく試したい」 | レシピ提案ボットのような、シンプルな1問1答から始める |

### パターン2: 専門知識対応型（高機能）

二つ目のパターンは、**make ＋ Dify ＋ ナレッジベース** を組み合わせる方法です ✅。紹介されている例では、美容業界のQ&Aボットが実装されており、データベース（ナレッジベース）と連携することで、より専門性の高い回答ができるようになっています ✅。

こちらは「設定は少し複雑になるが、専門的な質問に答えられる」という位置づけです。パターン1が「1問1答の簡易ボット」だとすれば、パターン2は「業種特有の知識を踏まえた相談窓口」に近いイメージです。

### 関連の実例（タイトルからの推定・本文未読）

「Lステップ連携ラボ」では、これ以外にも次のような実例記事のタイトルが確認できています。ただし本記事執筆時点では記事本文までは読み込めていないため、内容はタイトルからの推定にとどめます 🔶。

- 「就活支援AIチャットボット（ES自動作成・添削、適性診断）」

- 「回数制限付きAIチャットボットの実装」

また、参考として言及されている成果として、BtoB企業でメッセージ作成工数が週10時間からほぼゼロになった、資料請求率が3.4倍になった、飲食店で予約数が2.7倍になった、といった数字が挙げられています。ただしこれらは一次記事を直接読み込んだ上での確認ではなく、検索結果ベースの参考情報である点に注意してください 🔶。具体的な数字を自分の発信に転用する際は、必ず一次情報（note記事本文）に当たって裏取りしてから使うことをおすすめします。

### コピペで試せる考え方の型

実際にmakeやDifyを触る前に、まず「何を、どう頼むか」を日本語で整理しておくと迷いません。


## 5. LINE Harnessとは — 0円のオープンソースCRM

ここからは、2026年に入って急速に注目を集めている新興ツール、**LINE Harness**を紹介します。

LINE Harnessは、LINE公式アカウント運用のための**完全オープンソースのCRM**です ✅。開発者は野田修一氏（X: @ai_shunoda、GitHub: @Shudesu）で、「The Harness」というプロジェクトシリーズの第一弾として公開されています ✅。

- 公式サイト: https://shudesu.github.io/line-harness-oss/
- GitHubリポジトリ: https://github.com/Shudesu/line-harness-oss
- ライセンス: MIT（商用利用・改変・再配布が自由）✅

### 料金 — 完全無料という破壊力

LINE Harness最大の特徴は、**完全無料**であることです ✅。Cloudflare Workers上で動作し、友だち5,000人までは実質0円で運用できます ✅。5,000人を超えるとCloudflareの利用料が発生しますが、これはあくまでインフラを提供するCloudflareへの支払いであり、LINE Harness自体への課金ではありません ✅。

Lステップが月額5,000円〜32,780円程度 🔶 という価格帯であるのに対し、LINE Harnessは「自分でセットアップする手間」と引き換えに、ツール利用料そのものをゼロにしている構図です。

### バージョンと開発スピード

GitHubのREADME記載によれば、本記事執筆時点（2026年7月）の現行版は**v0.14.1** ✅ です。開発者は以前のXポストで「v0.4.0でMCPサーバー対応」を発表しており、そこから急速に機能拡張が続いている点は留意しておくべきです 🔶。オープンソースかつ個人開発ペースのプロジェクトであるため、機能や仕様は今後も変わっていく可能性があります。

### セットアップ — 1コマンド・約5分

LINE Harnessのセットアップは、`npx create-line-harness` という**1コマンド**で完結します ✅。このコマンドを実行すると、次の作業が自動で行われます。

- Cloudflareアカウントとの連携
- D1データベース（Cloudflareが提供するSQLiteベースのDB）の作成
- Worker（バックエンド）／管理画面のデプロイ
- LIFF（LINEアプリ内で動くWebページの仕組み）の自動作成

所要時間の目安は**約5分** ✅ とされています。必要な環境は次の3つです ✅。

- Cloudflareアカウント（無料枠で可）
- LINE公式アカウント＋Messaging APIチャネル
- Node.js 22以上／pnpm

「サーバーを借りて、環境構築して……」という従来型の"エンジニアの作業"を、コマンド1つに圧縮しているのがポイントです。

### 主な機能

LINE Harnessが備える機能は、次のように多岐にわたります ✅。

- ステップ配信（分単位の遅延制御）
- ブロードキャスト配信（500人超は自動キュー化）
- リッチメニューの動的切り替え
- LIFFフォーム
- IF-THEN自動化（7種トリガー×6種アクション）
- Stripe決済連携
- クリック追跡による自動タグ付け
- 予約管理（Googleカレンダー連携）
- スコアリング・タグ管理
- 友だち重複検出
- マルチアカウント管理
- BAN検知＆自動フェイルオーバー

さらに広告連携として、Meta（Facebook/Instagram）・X・Google Ads・TikTokのコンバージョン計測にも対応しています ✅。これは、本記事の執筆環境で実際にLINE HarnessのMCPサーバーに接続し、ツール仕様として確認済みの情報です。なお、個別アカウントの実データは一切参照していません。

### 技術スタックと姉妹プロジェクト

技術面では、Worker側にHono、Web（管理画面）側にNext.js 15、データベースにCloudflare D1（SQLite）を採用したTypeScript製プロジェクトです ✅。公式SDKとして`@line-harness/sdk`も提供されています ✅。

また「The Harness」シリーズとして、X向けの「X Harness」（github.com/Shudesu/x-harness-oss）、Instagram向けの「IG Harness」（github.com/Shudesu/ig-harness-oss）という姉妹プロジェクトも同じ開発者の手で公開されています ✅。LINEだけでなく、SNS運用全体を「オープンソース＋AIエージェント」で置き換えていく構想がうかがえます。

### セキュリティ面での注意点

「オープンソース＝危険」というイメージを持つ人もいるかもしれませんが、実際には逆で、世界中のエンジニアがコードを検証できる透明性こそが強みだという指摘があります 🔶（出典: kontaworks.com/line-harness-security/）。とはいえ、次のような注意点は押さえておく必要があります。

- Cloudflareアカウントの乗っ取り対策（二段階認証の設定）
- APIキー漏洩の防止（GitHubへの誤コミット・誤公開など）

また、ある有料note記事の無料公開部分では、「技術力のある事業者向けであり、完全な非エンジニアが一人で導入するのは難しい」という指摘もなされています 🔶（出典: note.com/chaen_channel/n/ndebdc9d60265）。無料・オープンソースであることと、「誰でも簡単に使える」ことは、必ずしもイコールではない点に注意してください。

## 6. LINE Harnessを使う方法 — Claude Codeで自然言語運用

LINE Harnessが単なる「無料のLステップ代替」で終わらないのは、**MCPサーバーを標準搭載**しているからです ✅。MCP（Model Context Protocol）とは、AIエージェントが外部のツールやサービスを「道具」として直接操作できるようにする仕組みのことです。これにより、Claude CodeのようなAIエージェントから、LINE公式アカウントの機能を自然言語だけで操作できるようになります ✅。

つまり、これまで管理画面を開いてクリックして……という作業が、チャットで「友だち数教えて」と話しかけるだけで完結するようになったということです。

管理画面を1つずつ開いて操作していた作業が、丸ごと「AIエージェントへの依頼」に置き換わっているイメージです。

### セットアップから自然言語運用までの流れ（比喩で理解する）

初めての人向けに、比喩を使って流れを整理します。LINE Harnessのセットアップは「新しいスタッフを1人雇って、机と道具一式を用意する」ようなものです。`npx create-line-harness` を実行すると、その"新人スタッフ"の机（Cloudflare Workerとデータベース）と、名刺（LIFF・管理画面）が自動的に用意されます。

そして、Claude CodeなどのAIエージェントとMCPサーバーを接続すると、その"新人スタッフ"に日本語で仕事を頼めるようになります。「友だち数教えて」と言えば集計してくれますし、「VIPに配信して」と言えば、あなたの代わりに配信作業をしてくれる——これが、LINE HarnessをAIエージェントで運用するということの実態です。

具体的な流れは、次の4ステップで理解すると分かりやすいでしょう。

1. **道具を揃える**: Cloudflareアカウント・LINE公式アカウント（Messaging APIチャネル）・Node.js環境を用意する ✅
2. **1コマンドでセットアップ**: `npx create-line-harness` を実行し、約5分待つ ✅
3. **AIエージェントと接続**: Claude CodeなどからMCPサーバーとして接続する ✅
4. **話しかけて運用**: 「友だち数教えて」「VIPに配信して」など、日常語で指示する ✅

このステップの中で、多くの人が身構えてしまうのが2番目の「1コマンドでセットアップ」の部分でしょう。ですが、これは「難しいことを1コマンドに圧縮してもらっている」だけであり、コマンドの中身を理解する必要はありません。料理でいえば、下ごしらえ済みのミールキットを電子レンジで温めるようなものです。

## 7. Lステップ vs LINE Harness — どちらを選ぶか

ここまで見てきたLステップとLINE Harness、それぞれの特徴を1つの表に整理します。

| 観点 | Lステップ | LINE Harness |
|---|---|---|
| 料金 | 月額5,000円〜32,780円程度 🔶 | 実質0円（自己ホスティング）✅ |
| API公開 | 非公開が基本 🔶 | 100以上のREST APIを完全公開 ✅ |
| AIエージェント対応 | 公式対応なし。Webhook経由でChatGPT/Difyなどと間接連携するのが主流（make等のノーコードツール併用）✅ | MCPサーバー標準搭載、Claude Codeから直接操作可能 ✅ |
| 運用のしやすさ | 非エンジニアでもGUIで完結。サポート・実績が豊富 🔶 | 技術的セットアップと自己責任の比重が高く「技術力のある事業者向け」🔶 |
| ライセンス・拡張性 | クローズドな有料サービス | MIT license、商用利用・改変・再配布自由 ✅ |

### 使い分けの目安

数字や機能だけを見るとLINE Harnessが圧倒的に有利に見えるかもしれませんが、実際には「どちらが優れているか」ではなく「自分の状況にどちらが合っているか」で考えるべきです 🔶。筆者の整理として、次のような目安を提示します（断定はしません）。

- **非エンジニアで、手厚いサポートを重視するなら → Lステップ**。GUI操作で完結し、代行業者や解説コンテンツも豊富にあるため、困ったときに頼れる先が多いのが強みです。
- **コストを抑えたい・技術力があり自由度を求めるなら → LINE Harness**。無料である代わりに、セットアップやAPIキー管理などをある程度自分（またはチーム）で担う必要があります。
- **すでにLステップで運用中の場合**、無理に乗り換える必要はありません。まずはWebhook経由でのAI連携（第4章）から試すのが現実的です。
- **これから新規にLINE運用を始める・コストをかけずに試したい場合**、LINE Harnessの無料枠から試してみるという選択肢が増えました。

繰り返しになりますが、これは「どちらか一方が正解」という話ではなく、2026年時点で選択肢が広がったこと自体が重要なポイントです。以前は「Lステップ一択」という状況に近かった市場に、無料かつAIエージェントとの相性がよい選択肢が登場した——この構図の変化こそが、本記事で伝えたい最大のポイントです。

## 8. もう一歩進んだ使い方

ここからは、開発者・技術者向けにもう一歩踏み込んだ話を軽めに触れておきます。本筋（初心者〜中級者向けの解説）を重くしすぎないため、詳細は最小限にとどめます。

### MCPサーバーという設計思想

LINE HarnessがMCPサーバーを標準搭載しているという事実は、単に「AIから操作できて便利」という以上の意味を持ちます。MCP（Model Context Protocol）はAnthropicが提唱するオープンな規格で、AIエージェントと外部ツールを接続するための「共通の話し方」を定義するものです。LINE Harnessがこの規格に対応しているということは、Claude Code以外の対応クライアントからも、原理的には同じように操作できる可能性があるということです。

### API連携という選択肢

LINE Harnessは100以上のREST APIを完全公開しています ✅。MCPサーバー経由の自然言語操作だけでなく、自社の他システム（在庫管理・予約システム・独自の顧客管理ツールなど）と直接つなぎ込むことも可能です。公式SDK「@line-harness/sdk」も用意されているため、TypeScriptでの独自開発にも対応しやすい設計になっています。

### 姉妹プロジェクトとの組み合わせ

前述の通り、同じ開発者は「X Harness」「IG Harness」という姉妹プロジェクトも公開しています ✅。これらを組み合わせることで、LINEだけでなくX・Instagramを含めたSNS運用全体を、AIエージェント経由で一元管理する構想も見えてきます。ただし、これらの組み合わせ運用についての具体的な事例やベストプラクティスは、本記事執筆時点ではまだ情報が少なく、今後の発信を注視する必要があります ⚠️。

このあたりは開発者コミュニティの動きが速い領域のため、最新情報は公式GitHub・開発者本人のXアカウントを直接確認することを強くおすすめします。
